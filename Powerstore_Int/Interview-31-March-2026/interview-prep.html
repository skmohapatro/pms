Core Java — Frequently Asked

Explain HashMap internals — how does it handle collisions?

HashMap uses an array of Node<K,V> buckets. Index = hash(key) & (n-1).

  • Collision handling: Linked list at each bucket (Java 7). In Java 8+, when a bucket exceeds 8 nodes, it converts to a Red-Black Tree (O(log n) lookup).
  • Load factor: default 0.75 — rehashes (doubles capacity) when threshold exceeded.
  • Key contract: equals() and hashCode() must be consistent.
Tip: Mention ConcurrentHashMap for thread safety — uses segment/bucket-level locking (Java 8+: CAS + synchronized on node).
What are the key differences between Java 8, 11, 17, and 21?
VersionKey Features
Java 8Lambdas, Streams, Optional, default methods, new Date/Time API
Java 11var in lambdas, HttpClient API, String methods (isBlank, strip), single-file execution
Java 17Sealed classes, pattern matching for instanceof, records, text blocks
Java 21Virtual threads (Project Loom), record patterns, sequenced collections, pattern matching for switch
Remember: Virtual Threads (Java 21) are game-changing for high-throughput I/O applications — lightweight, managed by JVM, not OS.
Explain the Java Memory Model — Stack vs Heap, GC generations
Stack (per thread)
  • Stores local variables, method frames
  • LIFO, auto-cleaned on method exit
  • StackOverflowError if too deep
Heap (shared)
  • All objects live here
  • Young Gen (Eden + S0/S1) → Minor GC
  • Old Gen → Major GC
  • Metaspace (class metadata, since Java 8)

GC Algorithms: G1 (default since Java 9), ZGC (low latency), Shenandoah. G1 divides heap into regions, does concurrent marking & evacuation.

What are Java Streams and how do parallel streams work?

Streams provide a declarative pipeline: source → intermediate ops → terminal op. They are lazy — intermediate operations only execute when a terminal operation is invoked.

List<String> result = employees.stream()
    .filter(e -> e.getSalary() > 50000)
    .map(Employee::getName)
    .sorted()
    .collect(Collectors.toList());

Parallel Streams use the ForkJoinPool.commonPool(). Use them for CPU-intensive ops on large datasets. Avoid for I/O-bound tasks or small collections (overhead > gain).

Pitfall: Parallel streams with shared mutable state → race conditions. Always use stateless operations.

Spring Framework & Spring MVC

Explain Spring IoC container, Bean lifecycle, and scopes

IoC (Inversion of Control): The container manages object creation and dependency injection. Two types: BeanFactory (lazy) and ApplicationContext (eager, preferred).

Bean Lifecycle:

  1. Instantiation → Populate properties → BeanNameAware → BeanFactoryAware → ApplicationContextAware
  2. @PostConstruct / InitializingBean.afterPropertiesSet() → init-method
  3. Bean ready for use
  4. @PreDestroy / DisposableBean.destroy() → destroy-method
ScopeDescription
singletonOne instance per container (default)
prototypeNew instance every time requested
requestOne per HTTP request (web)
sessionOne per HTTP session (web)
How does Spring MVC handle a request end-to-end?

Request flow:

  1. DispatcherServlet receives the HTTP request (front controller)
  2. Consults HandlerMapping → finds the right @Controller method
  3. HandlerAdapter invokes the controller method
  4. Controller returns a ModelAndView (or @ResponseBody → HttpMessageConverter)
  5. ViewResolver resolves logical view name → actual view
  6. View renders the response
Key: For REST APIs with @RestController, steps 5-6 are skipped — Jackson serializes directly to JSON.
What is the difference between @Component, @Service, @Repository, @Controller?

All are specializations of @Component and are detected by component scanning. The difference is semantic and some extra behavior:

AnnotationLayerExtra Behavior
@ComponentGenericBase stereotype
@ServiceBusiness LogicSemantic only (no extra magic)
@RepositoryData AccessTranslates persistence exceptions to Spring's DataAccessException
@ControllerWeb / MVCEnables @RequestMapping methods, view resolution
Explain Spring Boot auto-configuration and starters

Auto-configuration uses @ConditionalOnClass, @ConditionalOnMissingBean etc. to automatically configure beans based on classpath contents.

  • Starters are curated dependency sets (e.g., spring-boot-starter-web includes Tomcat, Jackson, Spring MVC).
  • spring.factories (pre-3.0) or META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports (3.0+) lists auto-config classes.
  • Your explicit beans always override auto-configured ones.

Servlets

How do Servlets work and how does Spring relate to them?

Servlet = Java class that handles HTTP requests inside a servlet container (Tomcat, Jetty).

Lifecycle: init() → service(doGet/doPost) → destroy()

Spring MVC's DispatcherServlet IS a servlet — it's the bridge between the servlet container and Spring's request handling.

Key Insight: Understanding servlets shows you know what's happening "under the hood" of Spring MVC. Mention Filter, Listener, HttpSession for bonus points.

How do you design a RESTful API — best practices?
  • Nouns not verbs: /users not /getUsers
  • Plural resources: /users/{id}/orders
  • HTTP methods: GET (read), POST (create), PUT (full update), PATCH (partial), DELETE
  • Status codes: 200 OK, 201 Created, 204 No Content, 400 Bad Request, 401 Unauthorized, 403 Forbidden, 404 Not Found, 409 Conflict, 500 Internal Error
  • Versioning: URI (/v1/), header, query param
  • Pagination: ?page=2&size=20 with total count in headers/body
  • HATEOAS: Include navigational links in responses
  • Idempotency: GET, PUT, DELETE should be idempotent
What is WSDL and how does SOAP communication work?

WSDL (Web Services Description Language) defines the SOAP service contract:

  • Types: Data types (XSD)
  • Messages: Request/Response structure
  • PortType: Operations available
  • Binding: Protocol and format (SOAP over HTTP)
  • Service: Endpoint URL

SOAP Envelope: Header (auth, routing) + Body (payload) + optional Fault (errors)

In Java: Use JAX-WS (@WebService, @WebMethod) or Spring WS with Jaxb2Marshaller.

🌐 SNMP & Networking

Why SNMP matters at Dell: Dell PowerStore, PowerScale, and other storage/infrastructure products use SNMP heavily for monitoring, alerting, and management of hardware and software components.
What is SNMP and how does it work?

SNMP (Simple Network Management Protocol) — protocol for collecting and organizing info about managed devices on IP networks.

Key Components:

ComponentRole
Manager (NMS)Central system that polls agents and receives traps
AgentSoftware on managed device that exposes data
MIB (Management Information Base)Hierarchical database of OIDs describing device properties
OID (Object Identifier)Unique identifier for each variable (e.g., 1.3.6.1.2.1.1.1 = sysDescr)

Operations:

  • GET — retrieve a specific OID value
  • GET-NEXT — walk through MIB tree
  • GET-BULK (v2c+) — efficient retrieval of large tables
  • SET — modify a value on the agent
  • TRAP — agent → manager unsolicited notification (async)
  • INFORM (v2c+) — acknowledged trap
SNMP Versions — v1, v2c, v3 differences?
Featurev1v2cv3
SecurityCommunity strings (plaintext)Community strings (plaintext)USM (auth + encryption)
AuthNoneNoneMD5 / SHA
EncryptionNoneNoneDES / AES
Bulk opsNoGET-BULKGET-BULK
InformNoYesYes
Always recommend SNMPv3 in production — it provides authentication, encryption, and access control. v1/v2c community strings are sent in plaintext!
Networking fundamentals — TCP vs UDP, DNS, HTTP/HTTPS
TCP
  • Connection-oriented (3-way handshake)
  • Reliable, ordered delivery
  • Flow control, congestion control
  • Use: HTTP, SSH, FTP
UDP
  • Connectionless
  • No guarantees, no ordering
  • Low overhead, fast
  • Use: DNS, SNMP, video streaming

SNMP uses UDP (port 161 for queries, port 162 for traps) — fire-and-forget for performance. INFORM provides reliability over UDP via acknowledgments.

Explain the Event Loop, Call Stack, and Task Queues
  1. Call Stack: Single-threaded, executes functions LIFO
  2. Web APIs: Browser handles async tasks (setTimeout, fetch, DOM events)
  3. Microtask Queue: Promises, queueMicrotask — higher priority
  4. Macrotask Queue: setTimeout, setInterval, I/O
  5. Event Loop: Drains call stack → drains ALL microtasks → picks ONE macrotask → repeat
console.log('1');
setTimeout(() => console.log('2'), 0);
Promise.resolve().then(() => console.log('3'));
console.log('4');
// Output: 1, 4, 3, 2
Closures, Hoisting, and 'this' keyword

Closure: A function that retains access to its lexical scope even after the outer function returns.

function counter() {
  let count = 0;
  return () => ++count; // closes over 'count'
}
const inc = counter();
inc(); // 1
inc(); // 2

Hoisting: var declarations (not initializations) are moved to top. let/const are hoisted but in Temporal Dead Zone (ReferenceError if accessed before declaration).

this: Depends on call-site. Arrow functions inherit this from enclosing scope. Regular functions: determined by how they're called (method, constructor, explicit bind).

Must-Know Design Patterns

Creational Patterns — Singleton, Factory, Builder

Singleton: One instance globally. In Java: use enum (best) or double-checked locking. Spring beans are singletons by default.

Factory Method: Define interface for creating objects; let subclasses decide which class to instantiate. LoggerFactory.getLogger().

Abstract Factory: Factory of factories — families of related objects.

Builder: Step-by-step construction. Great for objects with many optional parameters. StringBuilder, Lombok's @Builder.

Structural Patterns — Adapter, Decorator, Proxy, Facade
  • Adapter: Convert interface of a class into another. Arrays.asList() adapts array → List.
  • Decorator: Add behavior dynamically. Java I/O streams: new BufferedReader(new InputStreamReader(...)).
  • Proxy: Control access to an object. Spring AOP uses proxies for @Transactional, @Cacheable.
  • Facade: Simplified interface to a complex subsystem. Spring JdbcTemplate is a facade over JDBC.
Behavioral Patterns — Strategy, Observer, Template Method
  • Strategy: Define a family of algorithms, make them interchangeable. Comparator in Java.
  • Observer: Pub-sub model. ApplicationEventPublisher in Spring.
  • Template Method: Define skeleton in base class, steps in subclasses. AbstractController, JdbcTemplate.
  • Chain of Responsibility: Pass request along a chain. Servlet Filters, Spring Security filter chain.

🧪 TDD & Testing Strategy

TDD Cycle: RED → GREEN → REFACTOR
Write a failing test → Make it pass (simplest way) → Clean up the code
Explain TDD and its benefits
  1. RED: Write a test that defines desired behavior — it should fail
  2. GREEN: Write the minimum code to make the test pass
  3. REFACTOR: Clean up code while keeping tests green

Benefits: Better design (forces you to think about interface first), living documentation, regression safety net, confident refactoring, fewer bugs in production.

Mocking — Mockito essentials
// Create mock
UserRepository mockRepo = mock(UserRepository.class);

// Stubbing
when(mockRepo.findById(1L)).thenReturn(Optional.of(user));

// Verify interaction
verify(mockRepo, times(1)).save(any(User.class));

// Argument captor
ArgumentCaptor<User> captor = ArgumentCaptor.forClass(User.class);
verify(mockRepo).save(captor.capture());
assertEquals("John", captor.getValue().getName());

@Mock vs @Spy: Mock = fully fake object. Spy = real object with selective stubbing.

@InjectMocks: Automatically injects mocks into the class under test.

Authentication & Authorization — OAuth2, JWT, Spring Security

OAuth 2.0 Flows:

  • Authorization Code — most secure, for server-side apps
  • Client Credentials — machine-to-machine (no user)
  • PKCE — for SPAs and mobile apps (prevents code interception)

JWT (JSON Web Token): Header.Payload.Signature — stateless auth. Validate signature, check expiry, verify issuer/audience.

Spring Security: Filter chain architecture — SecurityFilterChain bean, @EnableWebSecurity, custom UserDetailsService, method-level security with @PreAuthorize.

Security in CI/CD pipelines
  • SAST (Static Analysis): SonarQube, Checkmarx — scan source code
  • DAST (Dynamic Analysis): OWASP ZAP — test running app
  • SCA (Software Composition): Snyk, Dependabot — check dependencies for CVEs
  • Container Scanning: Trivy, Aqua — scan Docker images
  • Secret Scanning: GitLeaks, TruffleHog — detect leaked credentials
  • SBOM: Software Bill of Materials — track all components

What are LLMs and how do they work (high level)?
  • LLM = Large Language Model — trained on massive text corpora to predict next tokens
  • Transformer architecture: Self-attention mechanism, processes sequences in parallel
  • Key concepts: Tokenization, embeddings, attention heads, context window
  • Fine-tuning vs Prompting: Fine-tune for domain-specific tasks, prompt engineering for general use
  • RAG (Retrieval-Augmented Generation): Combine LLM with external knowledge base for accurate, up-to-date answers

Enterprise use: Internal chatbots, document summarization, code assistance, log analysis, predictive maintenance for storage systems.

Responsible AI — what are the risks?
  • Hallucinations: AI generating plausible but incorrect information
  • Data Privacy: Don't send proprietary code/data to external AI services
  • Bias: Model outputs may reflect training data biases
  • Security: Prompt injection, adversarial attacks
  • IP Concerns: Copyright questions around AI-generated code
